# WhatToWear

一个小团队内部使用的“猜同事今天穿什么”Web 服务。它支持手机访问、成员注册登录、每日竞猜、动态赔率、实际着装记录、自动结算和奶茶积分榜。

## 功能

- 成员用昵称 + PIN 登录。
- 注册需要邀请码，管理员邀请码注册后拥有管理权限。
- 每日至少选择两个维度竞猜，截止前可修改。
- 内置上身、下身、外套、鞋子、发型、袜类等维度，并支持幽默选项。
- 管理员可在 Web 管理页维护竞猜维度、每个维度的选项、以及每个选项的人工初始可能性。
- 管理员可导入/导出配置，包含维度、截止时间、AI 赔率设置和 AI 概率结果。
- 管理员记录实际着装后，竞猜锁定并可结算。
- 结算规则：猜错者每人扣 1 份奶茶积分，猜中者按提交时锁定的赔率权重瓜分输家积分池，并生成兑现建议。
- 如果全员猜中，则启用全员池：每人投入 1 份积分，再按提交时锁定的赔率权重重新分配。
- 实际着装记录支持记录时间，默认取当前时间，也可手动调整用于补录历史；每个实际着装维度都可选择“未知”。
- 赔率模型：人工初始可能性 + 可选 AI 选项概率 + 实际着装历史的时间衰减贝叶斯更新；AI 与人工占比可在管理页调整。
- 普通成员可在赔率页查看 AI 对每个已有选项的预测概率和对应赔率。
- 多人猜中时按赔率权重瓜分猜错者积分。
- 全员猜中或全员猜错时当日作废，不产生积分变动。
- 竞猜时提供抽象实时效果图，颜色、款式、发型、鞋子等会随选择变化。
- 前端为响应式单页应用，兼容手机浏览器。

## 本地启动

```bash
python3 app.py
```

访问：

```text
http://127.0.0.1:8008
```

默认环境变量：

```text
PORT=8008
USER_INVITE_CODE=wear2026
ADMIN_INVITE_CODE=admin2026
SECRET_KEY=dev-change-this-secret
WHAT_TO_WEAR_DB=./data/what_to_wear.sqlite3
```

第一次注册的用户会自动成为管理员。生产环境建议显式设置 `SECRET_KEY`、`USER_INVITE_CODE` 和 `ADMIN_INVITE_CODE`。

## 赔率计算

系统会在用户提交竞猜时锁定当时赔率，后续管理设置或历史记录变化不会回改已经提交的竞猜赔率。

单个选项的初始概率来自人工维护的占比；启用 AI 后，会先调用 AI 生成每个已有选项的概率，再按管理页的 AI 占比混合：

```text
混合初始概率 = 人工概率 * (1 - AI占比) + AI概率 * AI占比
```

AI 请求会包含人工占比、目标日前一天的实际着装、管理页填写的天气预报/额外信息、以及历史着装记录。提示词会要求 AI 把人工占比当成衣柜/常见情况先验，避免简单认为“昨天穿过今天就不会穿”。

混合初始概率会继续经过历史着装的时间衰减贝叶斯更新。越近的记录权重越高，越久远权重越低；实际着装中记录为“未知”的字段不会参与概率训练。

多维竞猜会同时计算：

- 各维度独立后验概率的乘积。
- 完整组合在历史中的命中概率。

当历史中该组合样本越多，完整组合概率占比越高；样本少时更依赖各维度独立概率和初始先验。最终赔率约等于：

```text
赔率 = 1 / 最终预测概率
```

赔率有上限，避免极端小概率选项破坏结算。

## 结算规则

提交竞猜至少需要选择两个维度。选择了几个维度，就必须全部命中才算猜中；部分命中仍算猜错。

正常局：

```text
有赢家且有输家：猜错者每人 -1，猜中者按锁定赔率权重瓜分输家池。
```

全员猜错：

```text
当日作废，所有人 delta = 0。
```

全员猜中：

```text
每人投入 1 份积分形成全员池，所有人按锁定赔率权重重新分配。
净收益 = 分得金额 - 1
```

## 部署

### Docker Compose

复制环境变量模板并修改密钥：

```bash
cp .env.example .env
```

启动服务：

```bash
docker compose up -d --build
```

默认访问：

```text
http://127.0.0.1:8008
```

Compose 会把 SQLite 数据库持久化到 `what_to_wear_data` 数据卷，容器内路径为：

```text
/app/data/what_to_wear.sqlite3
```

如果要改宿主机端口，在 `.env` 中调整：

```text
HOST_PORT=8008
```

如果要导入旧数据库，可先停止服务，再把旧库复制进数据卷对应路径，文件名保持 `what_to_wear.sqlite3`。

### systemd

仓库内提供 systemd 服务模板：

```bash
sudo cp deploy/what-to-wear.service /etc/systemd/system/what-to-wear.service
sudo systemctl daemon-reload
sudo systemctl enable --now what-to-wear
```

默认监听 `0.0.0.0:8008`。
